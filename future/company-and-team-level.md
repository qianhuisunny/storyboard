# Future Feature: Persistent Context Layer

## Status: DEFERRED (Not for current sprint)

## Summary

Currently, Plotline generates a fresh Context Pack via research agents for every video. This works for one-time users but creates redundant work for repeat users (e.g., companies making multiple videos).

## Future Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    Context Hierarchy                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  [FUTURE] Company Context (persistent, user-uploaded)        │
│     │     - Brand voice & guidelines                         │
│     │     - Company background & positioning                 │
│     │     - Target audience personas                         │
│     │     - Product catalog & details                        │
│     │                                                        │
│     ▼                                                        │
│  [FUTURE] Campaign Context (per project, user-configured)    │
│     │     - Campaign goals & messaging                       │
│     │     - Specific product focus                           │
│     │     - Target segment for this campaign                 │
│     │                                                        │
│     ▼                                                        │
│  [NOW] Per-Video Research (generated by research agents)     │
│           - Topic-specific facts & terminology               │
│           - Supporting evidence & sources                    │
│           - Gap-filling for brief fields                     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## What This Enables

1. **Reduced redundancy**: Company info entered once, used across all videos
2. **Brand consistency**: Same voice/tone applied automatically
3. **Faster video creation**: Less research needed when context already exists
4. **Multi-modal expansion**: Same persistent context feeds learning paths, content strategy, certifications

## Implementation Considerations

### Data Model
```typescript
interface CompanyContext {
  id: string;
  name: string;
  brand_voice: {
    tone: string[];
    vocabulary: string[];
    avoid: string[];
  };
  company_background: string;
  products: ProductContext[];
  audiences: AudiencePersona[];
  created_at: Date;
  updated_at: Date;
}

interface CampaignContext {
  id: string;
  company_id: string;
  name: string;
  goals: string[];
  target_segment: string;
  messaging_pillars: string[];
  product_focus?: string;
}
```

### Onboarding Flow (Future)
1. New user signs up
2. Option A: Upload existing brand docs (PDF/Word) → AI extracts structured context
3. Option B: Guided interview → AI builds context from conversation
4. Option C: Skip for now → Full research mode (current behavior)

### Context Resolution Logic
```
When generating Context Pack:
  1. Check if Company Context exists → use as base
  2. Check if Campaign Context exists → layer on top
  3. Run research agents only for GAPS not covered by persistent context
  4. Merge all layers → final Context Pack
```

## Migration Path

Current Context Pack output already includes:
```json
{
  "context_source": "researched"  // ← This field enables future migration
}
```

Future states:
- `"context_source": "user_provided"` — From persistent context
- `"context_source": "hybrid"` — Mix of persistent + researched

Downstream agents (Brief Builder, Storyboard Director) won't need changes — they just consume the Context Pack regardless of source.

## Priority

**LOW for MVP / HIGH for enterprise customers**

Current one-time user flow works. Build this when:
- Signing enterprise customers who need multi-video consistency
- Expanding to other modalities (learning paths, content strategy)
- User feedback indicates repetitive context entry is painful
